与Mule合作开发团队。= 

这些书中的主题已经解决了您的本地工作站以及单个配置文件的单个项目。在一个团队工作时，Mule项目的规模将会增加，开发人员的数量会增加，并且必须在其他环境中运行，例如测试和生产。以下是一些可以使这种增长成为可能的做法。

将模块化Mule配置和应用程序的不同方法都是跨团队分工的机会，无论这些团队是在同一个总体项目上工作，还是在重用重复项目的不同项目上工作。
